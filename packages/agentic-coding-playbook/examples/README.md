# Examples

## English

Small, realistic examples that show the operative artifacts applied to a
concrete stack or workflow: a spec with evals, a task plan, a review report, a
PR description.

Concrete stack examples (for instance Symfony and Next.js) and GitHub Actions
examples are intentionally kept out of the first iteration to keep each change
reviewable. They will be added here in follow-up changes. Any CI example added
later will be clearly marked as an example, not a universally safe
configuration.

Until then, start from the artifacts they build on:

- [`../templates/`](../templates/)
- [`../policies/`](../policies/)
- [`../skills/`](../skills/)

## Deutsch

Kleine, realistische Beispiele, die die operativen Artefakte auf einen
konkreten Stack oder Workflow angewendet zeigen: eine Spec mit Evals, einen
Task-Plan, einen Review-Bericht, eine PR-Beschreibung.

Konkrete Stack-Beispiele (etwa Symfony und Next.js) und GitHub-Actions-Beispiele
bleiben bewusst aus der ersten Iteration heraus, damit jede Änderung reviewbar
bleibt. Sie werden hier in Folge-Änderungen ergänzt. Ein später ergänztes
CI-Beispiel wird klar als Beispiel gekennzeichnet, nicht als universell sichere
Konfiguration.

Bis dahin starte von den Artefakten, auf denen sie aufbauen:

- [`../templates/`](../templates/)
- [`../policies/`](../policies/)
- [`../skills/`](../skills/)
